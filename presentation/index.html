💡 Key Takeaway: The main cost driver is the LLM API, while infrastructure costs are relatively minor. LLM costs grow linearly with incidents, while infrastructure stays constant until we need to scale MCP servers.

⚡ Cost Optimisation Case Study

Starting Point
Baseline cost per diagnosis $0.385
Optimization Techniques
1. Selective Tool Access
Only expose needed tools (9K→4K tokens) -52%
2. Tool Caching
Cache tool descriptions ($0.3/M vs $3/M) -39%
3. Conversation Caching
Cache full context (124K→32 tokens!) -66%
Combined Result:
Original: $0.385 per diagnosis
Optimized: $0.067 per diagnosis
─────────────────────────
Total Savings = 83%
💡 Key Takeaway: At just 6.7¢ per diagnosis, you could run thousands of automated diagnoses for the cost of a single human engineer's hour.

💎 Beyond Cost: The Total Value Proposition

The true economic impact extends beyond direct cost comparisons:

  • ⚡ Response time: Autonomous agents can begin diagnosis immediately, reducing mean time to resolution (MTTR)
  • 🎯 Consistency: Agents apply the same thorough approach to every incident
  • 🧠 Knowledge retention: Unlike human teams that experience turnover and knowledge loss, agents maintain complete system knowledge
  • 🕐 24/7 availability: No off-hours, holidays, or vacation coverage concerns
  • 🚀 Team leverage: Human SREs can focus on high-value work instead of routine diagnostics
"Capability is only half the equation. As any engineering leader knows, the critical question that determines whether a technology moves from research to deployment is simple: Is it economically viable? LLM-based agents introduce a new economic model where each "cognitive operation" carries a direct, measurable cost. Every token processed, every reasoning step, every tool invocation adds to the bill. At scale, these costs can accumulate rapidly."
